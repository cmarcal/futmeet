# Docker Configuration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Docker support to `apps/api` with a `dev` profile (DB only) and a `deploy` profile (DB + auto-migration + compiled API).

**Architecture:** A 4-stage Dockerfile (deps → build → migrate → runtime) built with the monorepo root as context. Docker Compose profiles control which services start. The `migrate` service uses the `migrate` stage (keeps `tsx` + source) to run Knex migrations exactly as the local `db:migrate` script does. The `api` service uses the lean `runtime` stage and only starts after `migrate` exits successfully.

**Tech Stack:** Docker Compose v2 profiles, Node 22 Alpine, multi-stage Dockerfile, npm workspaces, tsx (migration runner), Knex (migrations), Fastify (API).

---

## Prerequisite: understand the `@futmeet/shared` resolution chain

`packages/shared/package.json` currently exports raw `.ts` files:

```json
"./types": { "import": "./src/types/index.ts" }
```

This works in local dev because `tsx` can execute `.ts` imports. But a compiled `dist/index.js` running under plain `node` would try to `import` a `.ts` file and fail. **Task 1 fixes this** before any Docker work begins.

## Task 2: Create `apps/api/Dockerfile`

**Files:**
- Create: `apps/api/Dockerfile`

The build context is always the **monorepo root** (two levels up from `apps/api/`). All `COPY` paths are relative to that root.

- [ ] **Step 1: Create `apps/api/Dockerfile`**

```dockerfile
# ---- Stage 1: Install all workspace dependencies ----
FROM node:22-alpine AS deps
WORKDIR /app
# Copy workspace manifests first (layer cache: only reinstall when lockfile changes)
COPY package.json package-lock.json ./
COPY packages/shared/package.json ./packages/shared/
COPY packages/tsconfig/package.json ./packages/tsconfig/
COPY packages/oxlint-config/package.json ./packages/oxlint-config/
COPY apps/api/package.json ./apps/api/
RUN npm ci

# ---- Stage 2: Build shared + API ----
FROM node:22-alpine AS build
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
# Copy shared package (source + tsconfig)
COPY packages/shared ./packages/shared
COPY packages/tsconfig ./packages/tsconfig
# Copy API source
COPY apps/api ./apps/api
# Build shared first so @futmeet/shared resolves to JS at API build time
RUN cd packages/shared && npx tsc
# Build API
RUN cd apps/api && npm run build

# ---- Stage 3: Migration runner (keeps tsx + full source) ----
FROM node:22-alpine AS migrate
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY packages/shared ./packages/shared
COPY packages/tsconfig ./packages/tsconfig
COPY apps/api ./apps/api
WORKDIR /app/apps/api
CMD ["node", "--import", "tsx/esm", "node_modules/knex/bin/cli.js", "migrate:latest", "--knexfile", "knexfile.ts"]

# ---- Stage 4: Lean production runtime ----
FROM node:22-alpine AS runtime
WORKDIR /app
# node_modules contains @futmeet/shared symlink → /app/packages/shared
COPY --from=deps /app/node_modules ./node_modules
# Make symlink target available: package.json (exports map) + compiled dist
COPY --from=build /app/packages/shared/package.json ./packages/shared/package.json
COPY --from=build /app/packages/shared/dist ./packages/shared/dist
# Compiled API
COPY --from=build /app/apps/api/dist ./apps/api/dist
WORKDIR /app/apps/api
EXPOSE 3001
CMD ["node", "dist/index.js"]
```

- [ ] **Step 2: Verify the `runtime` stage builds without errors**

Run from the **monorepo root**:

```bash
docker build -f apps/api/Dockerfile --target runtime -t futmeet-api-test .
```

Expected: All 4 stages complete with no errors. Final image is tagged `futmeet-api-test`.

- [ ] **Step 3: Verify the runtime image starts**

```bash

  -e PORT=3001 -e NODE_ENV=production -e API_KEY=test -e ALLOWED_ORIGINS=http://localhost:5173 \
  -p 3001:3001 futmeet-api-test
```

Expected: Fastify logs appear. Hit `http://localhost:3001/health` → `{"status":"ok"}`.

Stop the container (`Ctrl+C`). Clean up the test image:

```bash

```

```bash

git commit -m "feat(api): add multi-stage Dockerfile for deploy"
```

## Task 3: Update `apps/api/docker-compose.yml`

**Files:**
- Modify: `apps/api/docker-compose.yml`

Replace the entire file:

- [ ] **Step 1: Rewrite `apps/api/docker-compose.yml`**

```yaml
services:
  db:
    image: postgres:16-alpine
    profiles: [dev, deploy]
    environment:
      POSTGRES_DB: futmeet
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: password
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 5s
      retries: 5

  migrate:
    build:
      context: ../..
      dockerfile: apps/api/Dockerfile
      target: migrate
    profiles: [deploy]
    env_file: .env
    depends_on:
      db:
        condition: service_healthy
    restart: on-failure

  api:
    build:
      context: ../..
      dockerfile: apps/api/Dockerfile
      target: runtime
    profiles: [deploy]
    env_file: .env
    ports:
      - "3001:3001"
    depends_on:
      migrate:
        condition: service_completed_successfully
    restart: unless-stopped

volumes:
  pgdata:
```

## Appendix: Common commands reference

```bash
# Dev workflow — start DB, run API locally
cd apps/api
npm run docker:dev      # starts only db container
npm run db:migrate      # run pending migrations (from host)
npm run dev             # start API with hot-reload

# Deploy workflow — fully containerized
cd apps/api
# Edit .env: DATABASE_URL=...@db:5432/...
npm run docker:deploy   # starts db + migrate + api

# Tear down all containers
npm run docker:down

# On-demand migration run inside Docker
docker compose --profile deploy run --rm migrate

# Rebuild after code changes
docker compose --profile deploy up -d --build
```
